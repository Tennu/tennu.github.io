--- title: Response Values layout: document ---

Tennu events that have a channel property (including all commands) can return values telling Tennu to respond to the message with a certain message.

The return value can either be a string, an array of strings, or a response object.

The return value can also a promise of the previous values.

String

The simplest of the return values, this string is sent to the channel in a privmsg.

Using this, we can create a really simple echobot:

{% highlight javascript %} client.on('privmsg', function (privmsg) { return privmsg.message; }); {% endhighlight %}

Array of Strings

Just as Client.say can take an array of strings, you can also return an array of strings, and each string will be said to the channel in the order sent.

Response Object

{% highlight javascript %} { "message": required string | [string], "intent": 'say' | 'act', "query": boolean, "target": string } {% endhighlight %}
message
Either a string or array of strings of messages to output.
intent
Either 'say' or 'act'. By default, 'say'. Whether or not the message is an action or not.
target
Where to send the message to. Either a channel or a nickname.
